PLUMB

The Post-Screen Interface for AI Agents

Plumb is an MCP server that grants AI agents deep **LSP intelligence** and **safe filesystem access**. It eliminates raw-file context dumping by providing a semantic API for understanding and changing codebases.

System Architecture

One shared daemon. High-concurrency. Low-latency context.

Request Lifecycle

01

Singleton Daemon

A single long-running process serves all agent conversations. It maintains warm LSP connections to every workspace root, ensuring instant responses.

02

Session Isolation

Each MCP connection is an isolated session with its own read tracking, rate limits, and configuration overrides.

03

LSP-First Semantics

Plumb speaks protocol. It knows symbols, call hierarchies, and diagnostics. It doesn't guess; it queries the source of truth.

Safety Constraints

  • Read Tracking Optional strict mode requires a file be read before it can be edited, preventing "blind" mutations.
  • Optimistic Concurrency Edits can be fenced with expected_mtime to ensure the file hasn't changed since it was last read.
  • Dirty-State Guards Refuses to overwrite uncommitted git changes by default. Agents must explicitly request dirty_ok.
  • Rate Limiting Built-in sliding-window write budget protects against runaway loops and excessive context usage.
