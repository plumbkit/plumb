The problem

An agent reads whole files into its context — heavy, lossy, and blind to what a symbol means.

Plumb gives it a semantic API for the codebase instead: it queries the source of truth, and changes it without breaking the build.

01 — 03Principles

Three commitments, held plumb.

The discipline that lets a fleet of agents share one repository — safely.

A · Semantic

It queries, never guesses.

The agent asks the language server — the same source of truth your editor uses — instead of pattern-matching text and hoping.

  • rename_symbol — scope & type aware
  • diagnostics on every write
  • workspace symbol search
  • call & type hierarchies
B · Safe

Edits land, or they don't.

Industrial file safety, because you'll point more than one agent at the same tree and they must not corrupt each other.

  • atomic tmpfile → rename
  • per-path locks, all sessions
  • optimistic concurrency
  • multi-file transactions
C · Measured

Only what is needed.

Read the symbol, not the two thousand lines around it. Context is the scarcest resource an agent has.

  • file_outline — ~200 tokens
  • session_start — one call
  • SQLite/FTS5 topology index
  • durable per-repo memory

04How it works

One warm daemon, shared across every conversation.

Warm servers

Language servers stay warm between sessions — no re-indexing on a new chat. Only the first query is slow.

Shared, safe state

One daemon owns every connection, so per-path locks hold across chat windows. Concurrent agents can't corrupt each other.

LSP-native & resilient

External writes flow through didChangeWatchedFiles; if the daemon falls over, plumb serve respawns it and replays the handshake — invisibly.

06How it understands code

Two ways to see your code. Most tools give one.

Plumb runs a fast structural map and a precise, type-aware layer side by side — and falls back from one to the other automatically, so the agent is never blind.

The Map · Topology

Instant — even on broken code.

A living semantic graph of your whole repository, indexed into SQLite with FTS5 full-text search — the most-deployed database on earth. Ranked symbol search, blast-radius, and the exact tests to run after a change come back in milliseconds. No cold start. No language server. It keeps working while the code is mid-refactor and won't compile.

15+ languages & config formats · pure-Go tree-sitter, ships in the binary
The GPS · LSP

Precise. Type-aware.

When exactness matters, plumb speaks to the same language servers your editor trusts: real go-to-definition, every reference, scope- and type-aware rename. The answers aren't pattern-matched guesses — they come from the compiler's own model of your code.

Quick start

Three commands.

bash · install
# 1 · install the binary
$ go install github.com/plumbkit/plumb/cmd/plumb@latest
# 2 · connect your assistant
$ plumb setup claude-code
# 3 · optional — pin the root & enable project config
$ cd my-project && plumb init # auto-detected otherwise
