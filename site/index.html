01 / Capabilities

Thirty-three tools. One semantic layer.

Shared Daemon

One LSP process per workspace. Language servers stay warm between sessions — no re-indexing on every new chat.

Multi-Language

Symbols, definitions, call hierarchies, diagnostics, file search, and edits — across supported LSPs in a single MCP session.

Workspace-Aware

Drop a .plumb directory in your project root. Plumb detects it automatically, even across nested modules.

Semantic Edits

LSP-driven refactoring tools. Scope-aware edits, dry-run by default. Guarded by strict mtime concurrency checks.

Workspace detection — plumb finds the project root by walking up from the file URI: first looking for a .plumb/ directory (run plumb init at your project root), then falling back to the nearest language root marker (go.mod, pyproject.toml). The .plumb marker takes priority, preventing nested modules in testdata/ or vendor/ from being selected as the root.
